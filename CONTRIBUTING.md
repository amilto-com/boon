# Contributing to BOON/TOON

Thank you for your interest in contributing to BOON and TOON!

## Language Requirements

**All documentation, code comments, commit messages, and markdown files MUST be written in English.**

This includes:
- README files
- API documentation
- Code comments and JSDoc
- Commit messages
- Issue descriptions and discussions
- Pull request titles and descriptions
- Specification documents
- Example files and tutorials

### Rationale

English is the standard language for open-source software development, ensuring:
- Maximum accessibility for the global developer community
- Consistency across all project materials
- Easier collaboration and code review
- Better searchability and discoverability

## Code Style

- Use TypeScript for all source code
- Follow the existing ESLint configuration
- Write clear, self-documenting code
- Add JSDoc comments for public APIs

## Testing

- Write tests for all new features
- Ensure all existing tests pass before submitting
- Use Vitest for unit tests
- Aim for meaningful test coverage

## Commit Messages

Use clear, descriptive commit messages in English:

```
feat: add indefinite-length container support
fix: correct zigzag decoding for large integers
docs: update BOON v2 specification
test: add roundtrip tests for streaming decode
```

## Pull Requests

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes with clear commits
4. Ensure all tests pass
5. Submit a pull request with a clear description

## Specification Changes

Changes to the BOON or TOON specifications require:
- Clear rationale for the change
- Backward compatibility analysis
- Updated documentation
- Comprehensive test coverage
- Review by maintainers

## Questions?

Open an issue for questions or discussions about contributing.
